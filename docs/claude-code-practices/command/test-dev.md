# 测试环境全量回归测试

你是 API 回归测试执行器。读取测试用例文件，对**测试环境**执行全量覆盖测试。

> ⚠️ 使用前按你项目的实际情况修改下面的环境配置——占位符用 `your-domain` / 环境变量名统一写法，不要直接抄。

## 环境配置

- **HOST**: `https://test.your-domain.com`（换成你项目的测试环境域名）
- **API_KEY**: 从环境变量 `TEST_API_KEY` 读取（环境变量名按项目约定改）
- **测试用例文件**: `~/.claude/data/api-test-cases.md`（本仓库 `data/` 下的模板，装到全局后即在此路径；或替换成项目内 `.claude/data/api-test-cases.md`）
- **数据库（测试）**: SQLite 本地文件或从环境变量 `TEST_DB_DSN` 读取

## 执行步骤

### 1. 读取测试用例

读取 `~/.claude/data/api-test-cases.md`，解析所有接口和 JSON 请求体。

### 2. 按类型执行

**同步接口**：
- 端点：根据用例文件中的端点（如 `POST /api/v1/auth/login`、`GET /api/v1/users/profile`）
- 每个接口取 1 条用例执行
- 判定标准：状态码正确，响应包含预期字段

**异步任务**：
- 端点：`POST /api/v1/tasks`
- 每个任务类型取 1 条用例提交
- 判定标准：响应包含 `task_id` 且 `status` 不为空

### 3. 异步任务轮询

提交完成后等待，然后通过数据库查询 tasks 表验证：
- `status` = SUCCESS / FAILURE / 处理中
- `result_url` 或 `data` 是否包含有效 URL

### 4. 输出报告

生成测试报告到 `test-reports/测试环境回归报告-{日期}.md`（目录不存在时自动创建），包含：

| 接口 | 类型 | 状态 | 结果 | 备注 |
|------|------|------|------|------|

汇总：通过/失败/跳过数量。

## 注意事项

- $ARGUMENTS 中可传入筛选条件（如 `--api login` 只测某个接口，`--type async` 只测异步任务）
- 如果某接口返回配置错误说明环境未就绪，标记为"跳过"而非"失败"
- 并行执行无依赖的请求以提高效率
