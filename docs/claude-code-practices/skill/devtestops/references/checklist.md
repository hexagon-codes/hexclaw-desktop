# 各阶段检查清单与测试报告模板

## 覆盖场景清单（每次测试必须逐项检查）

- [ ] 正常流程（happy path）
- [ ] 异常/错误流程（每种错误类型独立验证）
- [ ] 降级/fallback 路径
- [ ] 边界输入（空值/零值/超长/非法/特殊字符/前后空格/大小写）
- [ ] 并发/重复提交（Loading 锁/幂等）
- [ ] 状态流转（合法状态 + 非法/未定义状态）
- [ ] 数据一致性（PRD↔代码 / Store↔UI / 缓存↔DB）
- [ ] 幂等/重复请求
- [ ] 超时/重试/部分失败
- [ ] 历史数据兼容/回归
- [ ] 安全（XSS/注入/权限绕过/会话管理/敏感信息泄露）

不适用的场景标注"不适用"并附原因，不能留空。

## 测试用例文档模板

```markdown

> PRD 来源：{prd_file_paths}
> 源码范围：{scanned_files}
> 测试分层方法论：基于 DevTestOps 提测准入基线

---
```

### 每条用例必须包含

- **编号**：{层级缩写}-{三位数字}（如 UT-001, CT-010, MT-001）
- **用例标题**：一句话描述
- **前置条件**（如有）
- **操作步骤**
- **预期结果**：必须具体可验证，不写"正常显示""符合预期"
- **数据来源**：PRD §章节 / 源码 file:line
- **优先级**：P0（核心流程）/ P1（重要分支）/ P2（边界和异常）

### PRD↔代码交叉比对规则

生成用例时必须做以下交叉比对，发现问题标记到一致性问题章节：

| 比对方向 | 发现 | 标记 |
|---------|------|------|
| PRD 定义了，代码已实现 | 正常 | 生成对应测试用例 |
| PRD 定义了，代码未实现 | 缺失 | `[PRD 未覆盖]` |
| 代码实现了，PRD 未定义 | 偏离 | `[代码超出 PRD]` |
| PRD 与代码矛盾 | 不一致 | `[不一致]` — 按两种预期各出一条用例 |
| Mock 数据与 PRD 不一致 | 契约违规 | `[Mock 缺失]` 或 `[Mock 不一致]` |

### 一致性问题验证方式标注规则

每个 GAP 必须标注 Claude Code 可用的验证工具：

| 验证工具 | 适用场景 |
|---------|---------|
| **Claude Code 使用 Grep 工具** | 静态代码扫描：字段使用、逻辑分支、标注规范 |
| **Claude Code 使用 Read 工具** | 读取源码确认实现细节 |
| **Claude Code 使用 Bash 工具执行测试框架** | 单元/模块级验证（Vitest/Jest/pytest/go test） |
| **Claude Code + Browser MCP** | UI 可见性验证（截图比对） |
| **Claude Code + Playwright MCP** | 浏览器自动化断言（元素/URL/DOM） |

## 测试代码生成规则

生成测试用例文档后，同时生成可执行的测试代码文件：

```
tests/
  unit/          # 单元测试（对应第一章）
  component/     # 组件测试（对应第二章）
  module/        # 模块测试（对应第三章）
  integration/   # 集成测试（对应第四章）
  api/           # API/Mock 契约测试（对应第五章）
  e2e/           # E2E 测试（对应第六章）
  security/      # 安全测试（对应第八章）
  testcases/     # 测试用例文档 + 测试报告
```

测试代码必须：
- 使用项目现有的测试框架和风格
- 每个 test 的名称包含用例编号（如 `test('UT-001: ...')`)
- 可独立执行，不依赖其他用例的执行顺序

### 红绿灯判定规则

| 状态 | 条件 |
|------|------|
| 🟢 绿灯 | 测试通过，行为符合 PRD 预期 |
| 🟡 黄灯 | 测试通过但发现非阻塞问题（建议修复 / 待产品决策 / 部分覆盖） |
| 🔴 红灯 | 测试失败 / PRD 与代码不一致且影响功能 / 安全风险 |
| ⚪ 未执行 | 需要特定环境或工具，当前未执行 |

### 整体判定规则

- 全部 🟢 → **🟢 绿灯**：可提测/可交付
- 有 🟡 无 🔴 → **🟡 黄灯**：核心功能正常，有非阻塞问题需跟进
- 有 🔴 → **🔴 红灯**：有阻塞问题，不建议提测/交付
