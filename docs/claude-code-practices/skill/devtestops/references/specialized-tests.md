# 专项测试触发规则与检查项

## 触发判断流程

Claude Code 使用 Grep 工具扫描变更文件，逐条匹配以下触发条件。命中则标记为"必须测"，未命中标记为"跳过"并附理由。

## 专项测试清单

### 1. E2E 测试

**触发条件**：涉及核心主链路、跨模块流程、登录/鉴权、下单/支付/扣费/任务提交等闭环。

**检查项**：
- [ ] 从真实入口走完整流程
- [ ] 前后端、服务间调用正常
- [ ] 状态流转完整（从起始到终态）
- [ ] 最终结果正确（数据落库、页面展示、消息发出）

**Claude Code 执行方式**：使用 Playwright MCP 或 Browser MCP 执行浏览器级别测试。

### 2. 接口/API 测试

**触发条件**：新增或修改接口、字段、错误码、分页、过滤、鉴权、兼容性。

**检查项**：
- [ ] 入参校验（必填、类型、格式、长度、非法值）
- [ ] 出参结构（字段完整性、类型正确性、枚举值合法）
- [ ] 状态码和错误码（200/400/401/403/404/500）
- [ ] 鉴权（未登录、Token 过期、越权）
- [ ] 幂等（重复请求不产生副作用）
- [ ] 分页和过滤逻辑
- [ ] 字段兼容（老字段不删、新字段有默认值）

**Claude Code 执行方式**：使用 Bash 工具执行 curl / httpie 测试；或使用 Bash 工具运行 API 测试框架。

### 3. 权限与安全

**触发条件**：涉及登录态、角色、资源归属、敏感字段、管理后台或外部入口。

**检查项**：
- [ ] 未登录访问受保护资源 → 拦截
- [ ] 越权访问（员工访管理员、用户 A 访用户 B 数据）→ 拦截
- [ ] 角色差异（不同角色看到不同内容/操作）
- [ ] 敏感字段不泄露（密码、Token、内部 ID 不出现在响应/日志/URL）
- [ ] 非法输入不执行（XSS、SQL 注入、命令注入）
- [ ] 高危操作有二次确认或限制

**Claude Code 执行方式**：
- 使用 Grep 工具扫描密码/Token 泄露：`grep -rn 'password\|token\|secret' --include='*.ts' --include='*.tsx'`
- 使用 Bash 工具执行安全测试用例

### 4. 并发/幂等/重试

**触发条件**：涉及扣减、提交、回调、状态流转、重复请求、超时重试。

**检查项**：
- [ ] 重复提交不重复生效（幂等）
- [ ] 重试安全（重试不产生额外副作用）
- [ ] 并发下状态不错乱（加锁/原子操作）
- [ ] 不出现脏写或多扣
- [ ] 超时后资源正确释放

**Claude Code 执行方式**：使用 Bash 工具执行 `go test -race` 或并发测试脚本。

### 5. 数据一致性

**触发条件**：涉及数据库、缓存、队列、任务状态、账务/配额/库存类数据。

**检查项**：
- [ ] 主表与明细一致
- [ ] 缓存与数据库一致
- [ ] 生产者与消费者一致
- [ ] 账务与业务状态一致
- [ ] 事务成功/失败/回滚后数据正确

### 6. 异步任务/队列

**触发条件**：涉及消息队列、定时任务、死信、重试、补偿、延迟处理。

**检查项**：
- [ ] 消息不丢失
- [ ] 不重复消费
- [ ] 顺序异常不导致业务错误
- [ ] 死信不堆积
- [ ] 补偿失败有告警
- [ ] 任务不卡死（有超时机制）

### 7. 配置与兼容性

**触发条件**：涉及配置开关、环境变量、灰度、老数据、老客户端或第三方字段变更。

**检查项**：
- [ ] 开关开启/关闭都符合预期
- [ ] 老数据可正常使用
- [ ] 旧版本客户端兼容
- [ ] 配置缺失时有兜底默认值
- [ ] 第三方字段变更不导致解析失败

### 8. 回滚与可观测性

**触发条件**：涉及上线发布、迁移脚本、灰度策略、关键日志、指标和告警。

**检查项**：
- [ ] 回滚方案可执行（步骤明确）
- [ ] 迁移脚本可逆
- [ ] 关键日志齐全（可追溯）
- [ ] 指标和告警能及时发现问题
- [ ] 灰度策略可控（可按比例/按用户放量）
